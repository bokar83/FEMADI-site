Guide d'utilisation du gestionnaire de contenu

Fondation Elhadj Madifing Diane (FEMADI) — Usage interne exclusivement
Version 1.1 Juin 2026 Sveltia CMS
🔐
Prérequis indispensable : vous devez disposer d'un compte GitHub autorisé par l'administrateur du site avant de pouvoir accéder au gestionnaire. Si vous n'avez pas encore de compte GitHub ou si votre accès n'a pas encore été accordé, contactez l'administrateur technique avant de commencer.
1

Accès au gestionnaire de contenu

▼

Le gestionnaire de contenu du site FEMADI est accessible directement depuis un navigateur web, sans installation de logiciel.

Adresse d'accès
🌎 https://femadi.com/admin/

Ouvrez cette adresse dans votre navigateur (Chrome, Firefox, Edge ou Safari recommandés). La page de connexion s'affiche.

Connexion via GitHub
1
Ouvrir la page d'accès

Rendez-vous sur https://femadi.com/admin/. Une page de connexion s'affiche avec le logo du gestionnaire.

2
Cliquer sur "Se connecter avec GitHub"

Le gestionnaire utilise votre compte GitHub pour vérifier votre identité. Aucun mot de passe CMS séparé n'est requis.

3
Autoriser l'application

Si GitHub vous demande d'autoriser Sveltia CMS à accéder à votre compte, cliquez sur Authorize. Cette autorisation est demandée une seule fois.

4
Accès accordé — tableau de bord

Une fois authentifié, vous accédez directement au tableau de bord du gestionnaire.

⚠️
Fenêtre pop-up bloquée ? Si la fenêtre de connexion GitHub ne s'ouvre pas, vérifiez que votre navigateur n'a pas bloqué les pop-ups pour le site femadi.com. Autorisez-les temporairement dans les paramètres de votre navigateur (barre d'adresse — icône pop-up bloqué).
ℹ️
À propos du versionnement : toutes les modifications sont enregistrées dans un système de versionnement (Git). Aucune erreur n'est définitive : si une modification incorrecte est publiée, l'administrateur technique peut restaurer la version précédente en quelques minutes.
2

Interface principale — Vue d'ensemble

▼

Une fois connecté, vous accédez au tableau de bord principal de Sveltia CMS. Deux rubriques de contenu sont disponibles dans le menu de navigation de gauche.

Les deux rubriques éditables
Rubrique dans le menu Ce que vous pouvez modifier Fichier concerné
Contact & Frais d'adhésion Téléphone, WhatsApp, adresse e-mail, et les trois niveaux de frais d'adhésion en francs guinéens (GNF) _data/site.json
Contenu du site / Textes (FR/EN) L'ensemble des textes visibles sur le site : Héro, Mission, Objectifs, Activités, Adhésion, Dons, Contact, Navigation, Pied de page — en français et en anglais _data/content.json
Navigation dans l'interface
  • Menu de gauche : cliquez sur une rubrique pour accéder à sa liste de contenus.
  • Bouton "Média" : accès à la bibliothèque d'images et de documents téléchargeables.
  • Bouton "Flux de travail" : affiche tous les contenus par statut (Brouillon, En révision, Publié).
  • Icône de profil (en haut à droite) : affiche le compte GitHub connecté et permet de se déconnecter.
💡
Conseil de navigation : en cas de doute sur la rubrique à modifier, référez-vous toujours à la colonne "Ce que vous pouvez modifier" du tableau ci-dessus. Les coordonnées et frais se modifient dans Contact & Frais d'adhésion ; tous les textes du site, dans Contenu du site / Textes (FR/EN).
3

Modifier les coordonnées et les frais d'adhésion

▼

Cette rubrique centralise les informations de contact de la fondation et les tarifs d'adhésion. Une modification ici se répercute automatiquement sur deux endroits du site.

ℹ️
Mise à jour automatique sur deux emplacements : quand vous modifiez un montant de frais d'adhésion dans cette rubrique, le changement s'affiche automatiquement dans la section Contact du site et dans la section Adhésion. Il suffit de modifier les données une seule fois ici — il n'y a pas d'autre endroit à mettre à jour.
Accéder à la rubrique
1
Cliquer sur "Contact & Frais d'adhésion" dans le menu de gauche

La liste des éléments de cette rubrique s'affiche.

2
Cliquer sur le seul élément disponible

Cette rubrique contient une seule entrée : les paramètres globaux du site. Cliquez dessus pour ouvrir le formulaire d'édition.

Champs disponibles

Format des montants : saisissez uniquement le chiffre, sans le symbole "GNF". Exemple : saisir 200 000 et non 200 000 GNF. Le symbole est ajouté automatiquement par le site.
Enregistrer les modifications

Une fois les champs modifiés, utilisez le bouton "Enregistrer" en haut du formulaire. La modification est sauvegardée en tant que brouillon. Pour qu'elle soit visible sur le site, suivez les étapes de la Section 6 : Sauvegarder et Publier.

4

Modifier les textes du site FR / EN

▼

La rubrique "Contenu du site / Textes (FR/EN)" regroupe l'ensemble des champs de texte organisés par section du site. Chaque texte existe en version française et en version anglaise.

⚠️
Toujours mettre à jour les deux langues. Chaque champ de texte existe en version française (_fr) et anglaise (_en). Si vous modifiez uniquement la version française sans mettre à jour l'anglais, les visiteurs qui consultent le site en anglais verront l'ancien texte. Modifiez toujours les deux langues dans la même session.
Accéder à la rubrique
1
Cliquer sur "Contenu du site / Textes (FR/EN)" dans le menu de gauche

La liste des groupes de texte s'affiche.

2
Cliquer sur l'unique entrée

Cette rubrique contient une seule entrée regroupant l'ensemble des textes du site. Cliquez dessus pour ouvrir le formulaire complet.

3
Faire défiler jusqu'au groupe souhaité

5

Gérer les images et les documents

▼

Le gestionnaire de contenu intègre une bibliothèque de médias permettant d'importer et d'organiser les images et documents utilisés sur le site. Tous les fichiers téléchargés sont stockés dans le dossier assets du site.

Télécharger une image ou un document
1
Cliquer sur "Média" dans le menu du gestionnaire

La bibliothèque de médias s'ouvre et affiche tous les fichiers déjà présents sur le site.

2
Cliquer sur "Télécharger" ou faire glisser votre fichier dans la fenêtre

Vous pouvez sélectionner un fichier depuis votre ordinateur, ou le faire glisser directement dans la zone d'importation (drag & drop).

3
Attendre la confirmation de téléchargement

Une icône ou un message confirme que le fichier est bien enregistré. Il apparaît ensuite dans la bibliothèque.

4
Utiliser le fichier dans un champ image

Dans les formulaires de contenu, cliquez sur un champ image pour ouvrir la bibliothèque et sélectionner le fichier que vous venez d'importer.

Bonnes pratiques de nommage
💡
Nommez vos fichiers avec des mots en minuscules séparés par des tirets, sans accents, ni espaces, ni caractères spéciaux. Exemple : femadi-conakry-2024.jpg. Un nom lisible facilite la maintenance du site à long terme.
Exemple incorrectExemple correct
Photo FEMADI Conakry (1).jpgfemadi-conakry-2024.jpg
Rapport Annuel 2025.pdfrapport-annuel-2025.pdf
logo_FEMADI_final_v2.pnglogo-femadi.png
Formats acceptés et taille recommandée
  • Images : JPG, PNG, WEBP. Taille recommandée : moins de 1 Mo par image pour un chargement rapide du site.
  • Documents : PDF uniquement pour les documents téléchargeables.
  • Taille maximale : contactez l'administrateur technique pour connaître la limite en vigueur.
6

Sauvegarder et publier vos modifications

▼

Toute modification dans Sveltia CMS passe par un flux de travail éditorial en trois étapes. Ce processus garantit que les contenus sont relus et validés avant d'être visibles par le public.

Le flux de travail en trois étapes
✎️
Brouillon
Modification enregistrée, non publiée
→
🔎
En révision
Soumis pour vérification
→
✅
Publié
Visible sur le site
Étape par étape
1
Modifier le contenu et enregistrer en brouillon

Après avoir rempli ou modifié les champs, cliquez sur le bouton "Enregistrer". La modification est sauvegardée en état Brouillon. Elle n'est pas encore visible sur le site public.

2
Passer l'entrée en révision

Dans la barre d'état du contenu (en haut du formulaire), changez le statut de Brouillon à En révision. Cela signale à l'équipe que le contenu est prêt à être vérifié avant publication.

3
Valider et publier

Une fois le contenu vérifié, passez le statut à Publié. Le système génère automatiquement une mise à jour du site. La modification apparaît sur le site en ligne dans les minutes qui suivent.

ℹ️
Délai de mise en ligne : après le passage au statut "Publié", le site peut prendre entre 1 et 5 minutes pour afficher les modifications. Rechargez la page du site public (Ctrl+F5 ou Cmd+Shift+R) pour voir le résultat.
✅
Versionnement automatique : chaque publication est enregistrée dans Git. Toute modification publiée par erreur peut être annulée par l'administrateur technique, qui peut restaurer une version précédente en quelques minutes.
Suivre l'état de vos contenus

Cliquez sur "Flux de travail" dans le menu principal du gestionnaire pour voir tous vos contenus classés par statut : Brouillon, En révision, Publié. Cliquez sur n'importe quel élément pour l'ouvrir et modifier son statut.

7

En cas de problème

▼
🛡️
Aucune erreur n'est permanente. Les modifications publiées sur le site sont toujours réversibles grâce au versionnement Git. Si une erreur est publiée par inadvertance, contactez immédiatement l'administrateur technique : la version précédente peut être restaurée en quelques minutes, sans perte de données.
Problèmes courants et solutions
Problème observéSolution recommandée
Impossible de se connecter ("Not authorized" ou "Accès refusé") Votre compte GitHub n'est pas encore autorisé. Contactez l'administrateur pour faire ajouter votre compte à la liste d'accès.
La fenêtre de connexion GitHub ne s'ouvre pas Vérifiez que les pop-ups ne sont pas bloqués par votre navigateur pour femadi.com. Autorisez-les temporairement dans les paramètres du navigateur.
Un champ enregistré ne s'affiche pas sur le site Vérifiez que le statut du contenu est bien Publié (et non Brouillon ou En révision). Rechargez le site avec Ctrl+F5.
Le site affiche un texte incorrect après publication Ne tentez pas de corriger en republiant plusieurs fois. Contactez l'administrateur technique pour une restauration de la version précédente via Git.
Un téléchargement d'image échoue Vérifiez le format (JPG, PNG, WEBP) et la taille du fichier (recommandé : moins de 1 Mo). Renommez le fichier sans accents, espaces ni caractères spéciaux, puis réessayez.
Le gestionnaire affiche une erreur inconnue Déconnectez-vous (icône de profil, en haut à droite), fermez le navigateur, puis reconnectez-vous. Si le problème persiste, contactez l'administrateur.
Un texte modifié en français s'affiche correctement mais le site en anglais reste inchangé Vous avez oublié de mettre à jour le champ anglais correspondant (_en). Retournez dans le formulaire et mettez à jour les deux versions.
Contact de l'administrateur technique

Pour toute question technique concernant le gestionnaire, l'accès au site, ou la restauration d'une version précédente, contactez l'administrateur technique du site FEMADI. Indiquez dans votre message :

  • La section ou le champ concerné
  • L'action effectuée
  • Le message d'erreur exact (une capture d'écran est très utile)
  • L'heure approximative de la modification
💡
Bonne habitude : avant de modifier un texte important (titre principal, texte de mission, frais d'adhésion), copiez le contenu actuel dans un fichier texte. Cela vous permet de le restaurer rapidement sans attendre l'administrateur si vous souhaitez revenir à la version précédente.
